# Описание проекта
Проект представляет собой приложение-мотиватор, заставляющее меньше сидеть в "Бесполезных" программах/приложениях, играх и тому подобном, занимать пользователя "Полезной" деятельностью. Планируется, что программа будет предоставлять пользователю несколько типов работы с программой

Бывает время, когда необходимо выполнить ту или иную работу, решить некоторую проблему, но при этом отвлекают развлекающие факторы, по типу скроллинга ленты или игр. 
Мой проект поможет решить эту проблему.

Основными пользователями будут люди, которым сложно сосредоточиться на одном деле, но при этом необходимо выполнить ту или иную работу.
Таким образом получаем, что пользоваться этим проектом будут студенты, обучающиеся (или их родители, установившие приложение на телефон своему ребёнку), некоторые взрослые люди, работающие из дома

Выставлять мой проект можно будет на различных сервисах, которые являются местом для распространения различного рода приложений, тк на данных площадках нередко находится целевая аудитория моего проекта.

Бизнес будет работать по схеме В2С, ведь основными пользователями приложения являются обычные покупатели. 

Особую популярность проект будет обретать во времена сессий и подготовок к экзаменам, т.е. летом и зимой, когда проходят основные сессии

# Сроки проекта

1 неделя (23 февраля - 1 марта): Создание монотонного, но рабочего кода для понимания работы алгоритма
2 неделя (2-8 марта): Исправление ошибок, добавление нескольких алгоритмов работы взаимодействия с пользователем
3 неделя (9-15 марта): Разбитие кода на модули, начало создания визуала приложения
4 неделя (16-22 марта): отладка разбития на модули, корректировка дизайна
5 неделя (23-29 марта): разработка окончательного дизайна, проверка работы модулей между собой
6 неделя (30 марта - 5 апреля): Добавление новых функций, исправление ошибок, параллельное тестирование проекта на мне.
7 неделя (6-12 апреля): анализ проведённого тестирования, корректировка планов работы, тестирование остальных алгоритмов работы
8-9 недели (13-26): Запасное время для неожиданных изменений в графике/незапланированных сложностей той или иной недели

Для создания моего проекта я выбрал использование Python, так как я с ним знаком больше, чем с остальными языками.
Пока что планируется использование следующих библиотек:
· Psutill Для анализа процессов и получения их названий
· time Для измерения времени, которое пользователь провёл в том или ином приложении
· AppOpener Для работы с самими процессами, их закрытием
· CustomTkinter - Базовая библиотека для создания простого дизайна (в будущем может быть заменена на более современный вариант дизайна)
· playsound используется для воспроизведения звуокв
